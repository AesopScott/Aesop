Introduction to Claude — The API & How It Works

Understand the Anthropic API and what it means to build with a large language model.

◆ Module 8 of 10

◆ Section 1

What Is an API

An API is an Application Programming Interface — a way for software to talk to other software. The Anthropic API lets your code send messages to Claude and receive responses, enabling you to build applications powered by Claude's intelligence.

◆ Section 2

Messages and Roles

The Claude API uses a messages format — conversations made up of alternating user and assistant turns. Understanding this structure is foundational. Everything you build will work within this conversation paradigm.

◆ Section 3

Tokens and Pricing

AI models process text as tokens — chunks of text roughly corresponding to word pieces. Understanding tokens helps you manage context limits, estimate costs, and optimize your prompts. Most models charge per token processed.

◆ Section 4

System Prompts

A system prompt is instructions you give Claude before the conversation begins — defining its role, behavior, constraints, and context. Mastering system prompts is the difference between a generic AI and a specialized tool built for your specific use case.

◆ Section 5

Temperature and Parameters

The API exposes parameters that control Claude's behavior — temperature (how creative or deterministic), max tokens, and others. Understanding these controls lets you tune Claude's output for different applications.
