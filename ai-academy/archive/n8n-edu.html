What Is n8n? Workflow Automation Explained

n8n (pronounced "nodemation") is an open-source workflow automation platform. Think of it as a visual programming environment where you connect apps, APIs, and services together using a drag-and-drop canvas — no code required for most tasks, but full code access when you need it.

Founded in 2019 by Jan Oberhauser, n8n has grown into one of the most popular self-hosted automation tools in the world, with over 400 integrations. Unlike cloud-only tools like Zapier, n8n runs on your own infrastructure — your data never leaves your servers.

Each automated process is called a workflow — a series of connected steps that execute in sequence.
n8n can replace Zapier, Make (Integromat), and large portions of custom glue code between your apps.
It has a free self-hosted version and a paid cloud version — for full control, self-hosting is the way to go.

Core Concepts — Nodes, Workflows & Triggers

Everything in n8n is built from three fundamental building blocks:

Nodes are the individual steps in a workflow. Each node does one thing: fetch data from an API, send an email, transform a value, query a database, call Claude, or run a piece of JavaScript. Nodes have inputs and outputs — data flows between them like water through pipes.

Triggers are special nodes that start a workflow. They fire on a schedule (cron), on a webhook call, when an email arrives, when a form is submitted, or on dozens of other events. No trigger = no automation.

Workflows are the complete connected graph of nodes. A workflow sits dormant until its trigger fires, then executes each node in sequence, passing data from one to the next.

Data in n8n flows as JSON — every node receives and outputs arrays of JSON items.
You can branch, merge, loop, and conditionally route data using built-in logic nodes.
Workflows can call other workflows — enabling modular, reusable automation components.

Install n8n — Local, Docker, or Cloud

n8n offers three setup paths depending on your needs. Choose the one that matches your comfort level and use case:

Option A — npx (fastest for trying it out): One command, no installation, runs immediately. Data doesn't persist between restarts — good for learning, not production.

# Requires Node.js 18+ npx n8n # Opens at http://localhost:5678

Option B — Docker (recommended for self-hosting): Persistent data, easy updates, production-ready.

docker run -it --rm \ --name n8n \ -p 5678:5678 \ -v ~/.n8n:/home/node/.n8n \ docker.n8n.io/n8nio/n8n
Option C — n8n Cloud: No setup at all. Go to n8n.io and sign up for a free trial. Best if you just want to learn without managing infrastructure.
For production self-hosting, pair Docker with a reverse proxy (nginx or Caddy) and a domain with SSL.

Navigating the Editor — Your First Workflow

Open n8n in your browser at http://localhost:5678. The interface has three key areas: the canvas (where you build workflows), the node panel (where you find integrations), and the execution log (where you debug runs).

Build your first workflow in under five minutes:

Click the + button to add your first node. Search "Schedule Trigger" and add it — this runs your workflow on a timer.
Add an HTTP Request node. Point it at a public API like https://api.coindesk.com/v1/bpi/currentprice.json.
Add a Set node to extract just the price value from the response.
Click Execute Workflow to run it manually. Watch the data flow between nodes in real time.
Congratulations — you just built an automation. Now activate it with the toggle at the top to let it run on schedule.

Credentials & Connecting Your Apps

Most useful workflows require authenticated connections to external services. n8n manages these through Credentials — securely stored connection configs you create once and reuse across all your workflows.

Go to Settings → Credentials → Add Credential. n8n supports OAuth2, API keys, basic auth, and service-specific flows for hundreds of apps. Common ones to set up first:

Gmail / Google Sheets: Use OAuth2 — n8n walks you through Google's authorization flow.
Slack: Create a Slack app, generate a bot token, paste it in. Takes 3 minutes.
Anthropic (Claude): Add your API key under the Anthropic credential type — unlocks AI nodes.
Airtable / Notion / GitHub: All use API keys — generate in each service's settings and paste in.
Credentials are encrypted at rest and never exposed in workflow exports — safe to share workflow JSON with teammates.

Data Transformation — Expressions, Code & the Set Node

Moving data between nodes is only half the job — you often need to reshape, filter, or enrich it along the way. n8n gives you three tools for this:

Expressions are inline formulas using double-curly syntax: {{ $json.email }} or {{ $now.toFormat('yyyy-MM-dd') }}. They work inside any node field and are the workhorse of data mapping.

The Set Node lets you explicitly define the output fields of a step — great for cleaning up messy API responses before passing them downstream.

The Code Node lets you write raw JavaScript (or Python) for anything expressions can't handle — complex transformations, loops, external library calls, or custom business logic.

// Example: Code Node — format a name const items = $input.all(); return items.map(item => ({ json: { fullName: `${item.json.firstName} ${item.json.lastName}`, email: item.json.email.toLowerCase() } }));

Building Real Workflows — Practical Automation Patterns

The fastest way to level up is to build workflows that solve real problems. Here are four high-value patterns ranked by complexity — each teaches a core n8n skill:

Email-to-Spreadsheet: Gmail trigger → filter by label → extract fields → append to Google Sheets row. Teaches: triggers, filtering, data extraction.
Daily Digest: Schedule trigger → fetch RSS feeds → merge results → format with a template → send Slack message. Teaches: schedules, HTTP requests, merging, messaging.
Form-to-CRM: Webhook trigger (from a form) → validate required fields → create contact in Airtable → send welcome email via Gmail. Teaches: webhooks, conditional logic, multi-app workflows.
AI Content Pipeline: New Google Sheets row → send content brief to Claude → receive generated text → write result back to sheet. Teaches: AI nodes, bidirectional data flow with spreadsheets.

AI-Powered Workflows — Connecting Claude & LLMs

n8n has first-class AI support. The AI Agent node lets you build autonomous reasoning loops — give it tools (like web search, database lookup, or sending emails) and a goal, and it figures out how to accomplish it step by step, exactly like OpenClaw but embedded inside your workflow.

For simpler tasks, the Basic LLM Chain node sends a prompt to Claude (or GPT, Gemini, etc.) and returns the response — one node, zero complexity.

Use the Anthropic Chat Model sub-node to connect Claude to AI Agent or LLM Chain nodes.
Give AI Agents memory using the Window Buffer Memory node — it maintains conversation context across steps.
AI Agents can use n8n's built-in tools: Calculator, Wikipedia, SerpAPI (web search), and any HTTP endpoint you define.
Combine n8n AI workflows with OpenClaw for a powerful stack: OpenClaw handles always-on agent tasks, n8n handles structured data pipelines and integrations.

Error Handling, Debugging & Production Hardening

A workflow that fails silently is worse than no workflow at all. Production-grade automations need proper error handling, logging, and alerting.

Error Trigger node: Add a separate "error workflow" that fires whenever any other workflow fails. Use it to send yourself a Slack alert with the error details.
Try/Catch pattern: Connect the error output of any node to a handler branch — log the failure, send an alert, and continue gracefully instead of stopping the whole workflow.
Execution log: Every run is stored in the execution history. Filter by status (success/error) and inspect the exact data at each node at the time of failure.
Wait node: Add delays between steps to respect API rate limits — especially important for bulk operations hitting third-party services.
Environment variables: Store API keys and config values as env vars rather than hardcoding them in workflows. Makes deployments portable and secrets manageable.

◆ Mastery

Custom Nodes, Sub-Workflows & Scaling

Once you've mastered built-in nodes, you can extend n8n in two powerful ways: custom nodes (build your own integrations as npm packages) and sub-workflows (modular, reusable workflow components called by other workflows).

Sub-workflows are particularly powerful for teams — build a "send notification" workflow once, then call it from 20 other workflows. Update it in one place, every workflow gets the fix.

Build custom nodes using the n8n node builder — it's a TypeScript SDK with strong documentation.
Use queue mode (with Redis + worker processes) for high-volume workflows that need horizontal scaling.
Share workflows with the community at n8n.io/workflows — thousands of pre-built workflows you can import and adapt.
Connect n8n to your PlayAGame.ai workflows: trigger content generation, manage player data pipelines, automate outreach sequences, or feed Claude agents structured game data.

You now have the foundation to automate almost anything. The question isn't what can you automate — it's what should you automate first.
