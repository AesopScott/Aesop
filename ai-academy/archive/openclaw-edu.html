◆ Concepts

What Is OpenClaw? Understanding AI Agents

OpenClaw isn't a chatbot — it's an autonomous AI agent. The difference is fundamental: a chatbot answers your questions, but an agent takes actions on your behalf, loops through tasks until they're done, and keeps running even when you're not watching.

Originally created by Austrian developer Peter Steinberger as "Clawdbot" in late 2025, it became one of the fastest-growing open-source projects ever — surpassing 300,000 GitHub stars in under four months. The core idea: your AI lives on your hardware, talks to you through apps you already use (Telegram, WhatsApp, iMessage), and connects to LLMs like Claude or GPT.

It doesn't contain its own AI — it's a framework that uses AI via APIs you provide.
Think of it as a persistent, always-on assistant running in the background of your life.
The "agentic loop" means it can plan, act, check results, and retry without you supervising each step.

How OpenClaw Works — Architecture Overview

OpenClaw has three core layers worth understanding before you touch a single file:

The Gateway is the control plane — a background process running on your machine that manages all incoming messages, routes them to the right agent, and handles tool execution. Everything flows through it.

The Agent (Pi) is the intelligence layer. It receives your messages, reasons using your chosen LLM, decides which tools or skills to invoke, and sends back responses. It runs in an "agentic loop" — acting, evaluating results, and retrying until a task is complete.

Skills are the capability layer — modular plugins stored as directories with a SKILL.md file. Skills define what your agent can actually do: send emails, control your browser, monitor a service, post to social media, and hundreds more built by the community.

The Gateway runs 24/7 as a background daemon — it wakes on a heartbeat even without your input.
Memory is stored as plain Markdown files on your machine — not in a cloud database.
The LLM (Claude, GPT, etc.) is the "brain" — OpenClaw just gives it hands and eyes.

◆ Setup

Choose Your Machine & Prepare Your Environment

Before installing anything, you need to make a critical decision: where will OpenClaw live? Because it runs continuously, has file access, and can execute commands, the machine you choose matters.

Best option: A dedicated device — a spare Mac Mini, Raspberry Pi, or a cheap cloud VPS (~$5/month). This keeps it running 24/7 and away from your primary files.

Acceptable for learning: Your main laptop — but understand the risks, keep its permissions limited, and plan to migrate later.

macOS and Linux are the smoothest experience. Windows works via WSL2 (Windows Subsystem for Linux).
Requirements: Node.js 20+, npm, and a terminal you're comfortable using.
Give your agent its own accounts — separate email, storage, and API keys. Don't hand over your primary credentials.
# Check your Node version — needs 20+ node --version # If you need to install/upgrade Node, use nvm: curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash nvm install 20 nvm use 20

◆ Setup

Install OpenClaw & Run Onboard

Installation is straightforward — OpenClaw ships with an interactive onboarding CLI that walks you through the entire setup step by step. Don't skip it; it handles workspace creation, channel setup, and initial configuration.

# Install OpenClaw globally via npm npm install -g openclaw # Run the interactive onboarding wizard openclaw onboard # After onboarding, start the gateway openclaw start # Check health status openclaw doctor
The onboarding wizard will prompt you to connect at least one channel (start with Telegram — it's the simplest).
openclaw doctor surfaces misconfigurations before they become problems. Run it often.
The Gateway dashboard opens at http://localhost:18789 by default — this is your control panel.

◆ Setup

Connect Your LLM & First Channels

OpenClaw needs a brain — an LLM API to power the agent's reasoning. You'll configure this in the .env file in your workspace. Claude (via Anthropic API) is a popular choice for its strong instruction-following.

# In your .env file (located in ~/.openclaw/workspace/) ANTHROPIC_API_KEY=sk-ant-... # Or use OpenAI OPENAI_API_KEY=sk-... # Configure your preferred model in config.yaml # model: claude-sonnet-4-20250514

For channels, Telegram is the recommended starting point — create a bot via @BotFather on Telegram, copy the token into your config, and you'll be messaging your agent within minutes.

Use OAuth connections over raw API keys where possible — more cost-predictable and more secure.
You can connect WhatsApp, Discord, iMessage, Slack, and 15+ more — but start with one and expand.
Send /status to your agent in Telegram to confirm it's alive and connected.

Understanding & Using Skills

Skills are OpenClaw's superpower. Each skill is a directory containing a SKILL.md file that tells the agent what the skill does, what tools it has, and when to use them. The agent reads skills automatically and decides when to invoke them.

OpenClaw ships with a built-in skill registry called ClawHub. With it enabled, your agent can search for and self-install new skills on demand — you can literally ask your agent "I want to be able to post to LinkedIn" and it will find and install the skill itself.

Skills live in three places: bundled (global), workspace-level, and user-level. Workspace skills override others.
Browse community skills at openclaw.ai — there are 100+ covering most common workflows.
The simplest way to add a skill: ask your agent to install it via ClawHub. It handles the rest.
Be cautious with third-party skills — Cisco's security team found skills performing data exfiltration without user awareness. Vet what you install.

Your First Real Workflow — Automating Something Useful

The best way to learn OpenClaw is to give your agent a real, recurring job. Start small and specific — something that saves you genuine time. Here are excellent first workflows ranked by complexity:

Daily briefing: Every morning at 9am, summarize your calendar, unread emails, and top news. One heartbeat task, no custom skill needed.
Email triage: Monitor inbox, flag priority messages, draft responses for your review. Needs Gmail skill — available on ClawHub.
File watcher: Watch a folder for new files, auto-categorize and rename them. Great intro to file system permissions.
Meeting notes: After each calendar event, prompt for notes and auto-file them in Notion or a local folder.

To set up a recurring task, edit your HEARTBEAT.md file in your workspace. The agent reads this checklist on every heartbeat cycle and acts on any items that are due.

◆ Building

Write Your Own Custom Skill

Once you've used built-in skills, the real power comes from writing your own. Custom skills let you connect OpenClaw to anything — your own APIs, internal tools, proprietary data sources, or creative workflows.

A skill is just a folder with a SKILL.md file and optional tool scripts. The SKILL.md describes what the skill does in plain language — the agent reads it and decides how and when to use it.

# Create a new skill directory mkdir ~/.openclaw/workspace/skills/my-skill cd ~/.openclaw/workspace/skills/my-skill # Create the SKILL.md — this is what the agent reads cat > SKILL.md << 'EOF' # My Custom Skill This skill [describe what it does]. ## When to use [Describe the trigger conditions] ## Tools - tool_name: [what it does] EOF
Write your SKILL.md descriptions clearly and specifically — the agent uses this to decide when your skill is relevant.
Tools can be shell scripts, Node.js modules, or Python scripts — OpenClaw executes them in isolated containers.
Ask your agent to help you write a skill — it can build the scaffold and test it in the same conversation.

◆ Advanced

Security, Permissions & Running Safely

OpenClaw's power comes with real security responsibility. Because it has shell access, browser control, and can send messages on your behalf — a misconfigured or compromised instance is a serious threat surface.

In January 2026, a critical WebSocket hijacking vulnerability (CVE-2026-25253, CVSS 8.8) was disclosed — any malicious website could steal your auth token and gain remote code execution on your machine. This has since been patched, but it illustrates why you must stay updated and run in isolation.

Use a dedicated machine. Run OpenClaw on a device you can physically disconnect — not your primary laptop.
Configure tool policies. Allow email reads; require approval before sends. Allow file reads; block deletions. Configure exec_approvals for high-risk actions.
Beware prompt injection. Malicious instructions can be embedded in emails or web content your agent reads — these can hijack its behavior.
Run openclaw doctor regularly. It surfaces risky DM policies and misconfigurations before they become incidents.
Stay updated. Run npm update -g openclaw weekly — the project moves fast and patches security issues quickly.

◆ Mastery

Multi-Agent Systems & What Comes Next

Once you've mastered a single agent, you can deploy multiple — each with a distinct role, isolated workspace, and specialized skill set. One agent plans and delegates. Others execute: coding, research, outreach, monitoring. You manage the team; they do the work.

OpenClaw supports multi-agent routing out of the box — you can route different channels or accounts to different isolated agents. A marketing agent. A developer agent. A personal assistant. All running in parallel, all accessible from your phone.

Each agent gets its own workspace with separate memory, skills, and configuration — fully isolated from others.
Agents can communicate with each other through structured handoffs — one agent's output becomes another's input.
The community around OpenClaw is growing fast — join the Discord, explore ClawHub, and share skills you build.
The project is now governed by an open-source foundation after its creator joined OpenAI — its future is community-driven.

You're not just using AI anymore — you're deploying it. That's a meaningful shift. Welcome to the agentic era.
