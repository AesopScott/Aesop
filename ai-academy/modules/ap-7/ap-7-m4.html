What Is a Coding Agent?

From autocomplete to autonomous programmer — the architecture of agents that read, write, and execute code.
How did AI move from suggesting a variable name to deploying production software?

In July 2023, Cognition AI's internal benchmarks showed their agent Devin could autonomously resolve GitHub issues — reading the issue, browsing the repository, writing a fix, running tests, and opening a pull request without human intervention. When the company published these results in March 2024, the demo of Devin completing a full Upwork freelance task stunned the software industry. Critics later questioned the cherry-picking of demos, but the underlying architecture — a long-context LLM paired with a bash shell, a code editor, and a browser — represented something genuinely new: an agent that treated software development as an end-to-end workflow, not a snippet-generation problem.

Anatomy of a Coding Agent

A coding agent is a system in which a large language model is granted tools — typically a file system, a terminal, a code execution sandbox, and sometimes a web browser — and is given a goal expressed in natural language. The model then plans a sequence of actions, executes them one by one, observes the output of each action (compiler errors, test results, stdout), and revises its approach accordingly. This observe-plan-act loop distinguishes a coding agent from a code-completion assistant.

GitHub Copilot (launched June 2021) is the archetype of a completion assistant: it predicts the next token given surrounding context. It does not run code, does not read error messages autonomously, and does not iterate. By contrast, a coding agent like OpenDevin (the open-source community recreation of Devin, released April 2024) maintains a workspace, keeps a scratchpad of progress, and issues shell commands to verify its own output.

The key architectural ingredients are: (1) a long-context model that can hold an entire repository in its context window; (2) tool-use APIs (function calling) that give the model structured access to the environment; (3) a persistent working directory so changes accumulate across steps; and (4) a feedback loop — the agent reads stdout, stderr, and test results and incorporates them into its next action.

Key Distinction

Code completion predicts the next token. Code generation produces a function on demand. Code agents plan, execute, observe, and iterate — often across dozens of tool calls — to satisfy a high-level goal.

The Tool Stack

The tools available to a coding agent determine what it can accomplish. The minimal viable set is: a bash shell (to run commands, install packages, move files), a file editor (to read and write source files), and a code runner (to execute scripts and capture output). More capable systems add a web browser (for documentation lookup and API key retrieval), a database client, and a diff/patch tool to produce clean pull requests.

Anthropic's Computer Use feature, announced October 2024 alongside Claude 3.5 Sonnet, extended this idea to the entire desktop: the agent sees screenshots, moves the cursor, and types into any application — including IDEs — rather than calling structured tool APIs. This blurred the boundary between a coding agent and a general computer-use agent.

Microsoft's integration of GPT-4 into Visual Studio Code via the Copilot Workspace preview (April 2024) took a different approach: the agent is scoped to a single repository and a single task (an issue or a feature request), but it can propose a multi-file edit plan before executing it, letting engineers review the plan before any code is written.

Key Terms
ReAct loopThe Reason + Act pattern (Yao et al., 2022) in which an agent alternates between producing a reasoning trace and executing a tool call, using the tool's output to inform the next reasoning step.
SandboxAn isolated execution environment — typically a container or VM — that prevents agent-generated code from affecting the host system or accessing unauthorized resources.
SWE-benchA benchmark released by Princeton NLP (2023) that measures whether an agent can resolve real GitHub issues from 12 popular Python repositories, judged by whether the official test suite passes.
Long-context windowThe number of tokens an LLM can process in a single forward pass. Claude 3 (2024) supports 200K tokens, enough to hold roughly 150,000 lines of code simultaneously.
SWE-bench Progress

When SWE-bench launched in 2023, the best systems resolved about 3% of issues. By mid-2024, top agents (Devin, SWE-agent from Princeton, AutoCodeRover) were resolving 13–20% of issues. By early 2025, leading systems exceeded 40% — a 13× improvement in roughly 18 months, driven almost entirely by better agent scaffolding rather than larger base models.

The progress on SWE-bench illustrates a recurring theme in AI agents: raw model capability matters, but the scaffolding — how the agent decides which tool to call, how it recovers from errors, how it manages long task horizons — often matters more. Scaffolding & Execution Engines

The orchestration layer that turns a capable LLM into a reliable software engineer.
Why does the same base model perform so differently depending on how it is scaffolded?

In April 2024, Princeton NLP researchers released SWE-agent, an open-source coding agent that outperformed Devin on the full SWE-bench test set despite using only GPT-4 Turbo — a commercially available model anyone could access. The key innovation was an Agent-Computer Interface (ACI): a custom set of shell commands (open, scroll, search, edit, submit) designed specifically for navigating code repositories. The researchers found that the quality of this interface had more impact on benchmark performance than model choice. A poorly designed interface caused the model to lose track of its location in a file, repeat failed edits, or get stuck in loops. The right interface kept the agent oriented and reduced wasted actions by roughly 40%.

What Scaffolding Does

Scaffolding refers to the external code that manages the agent's execution loop: deciding when to call the LLM, which tools to expose, how to format tool results, how to detect task completion or failure, and how to recover from errors. The LLM itself only ever sees a text prompt; the scaffolding determines what goes into that prompt and what happens with the output.

The simplest scaffolding is a while-loop runner: call the LLM, parse its output for a tool call, execute the tool, append the result to context, repeat. More sophisticated scaffolding adds: plan-and-execute (first ask the model to produce a plan, then execute each step separately), reflection (a second LLM call that critiques the agent's last action before it is committed), and memory management (summarizing old context when the window fills up).

LangChain (released October 2022) and LlamaIndex popularized plug-and-play scaffolding libraries. Microsoft's AutoGen (September 2023) introduced multi-agent scaffolding: one agent writes code, a second agent runs tests and reports back, a third agent reviews the diff. These specialized roles allow longer, more complex coding tasks to be decomposed and parallelized.

Princeton Finding

SWE-agent's ACI reduced unnecessary file-navigation actions by ~40% compared to a naive bash-access scaffold. This single design decision — giving the agent purpose-built commands for code exploration — accounted for the majority of its SWE-bench performance gain over prior systems using the same base model.

Execution Engines: Sandboxes & Containers

A coding agent that can run arbitrary code is a security liability unless properly isolated. The dominant approach is containerization: run the agent's workspace in a Docker container with no network access, a read-only mount of the source repository, and resource limits on CPU, memory, and wall-clock time. The container is discarded after each task, ensuring state does not leak between runs.

E2B (launched 2023) commercialized this idea as a cloud API: give it a code snippet, get back stdout, stderr, and file diffs from a fresh microVM that boots in under 200ms. OpenAI's Code Interpreter (introduced in ChatGPT in July 2023) uses a similar per-session sandbox — a Jupyter kernel isolated in a container — that resets when the conversation ends. By the end of 2023, Code Interpreter had become one of ChatGPT's most-used features, primarily for data analysis tasks.

The failure mode to watch is sandbox escape: an agent (or maliciously crafted input) that coerces the LLM into writing code that exploits a container misconfiguration to reach the host system. Responsible deployment requires defense-in-depth: cgroups resource limits, seccomp syscall filtering, no host-network bridging, and read-only file mounts beyond the working directory.

Key Scaffolding Patterns
Plan-and-ExecuteThe agent first produces a numbered plan in natural language, then executes each step with a separate LLM call. Reduces mid-task drift and makes plans auditable.
Reflection / CriticA second LLM call that evaluates whether the last action achieved its intended effect, and suggests a correction if not. Used in OpenAI's o1 reasoning model and Reflexion (Shinn et al., 2023).
Agent-Computer Interface (ACI)A curated, purpose-built command set designed for LLM-as-agent use, as opposed to exposing raw bash. Reduces the action space and improves reliability.
Multi-agent scaffoldingMultiple LLM instances with distinct roles (coder, tester, reviewer) that communicate through structured message passing. Introduced at scale by Microsoft AutoGen (2023).
AutoGen's 2023 Multi-Agent Benchmark

In Microsoft's initial AutoGen paper, a two-agent setup (coder + executor) solved 69% of HumanEval coding problems that a single-agent setup failed on. Adding a critic agent for reflection pushed that to 77%. The marginal cost of each additional agent is roughly one extra LLM call per step — a small price for large reliability gains on complex tasks.

Understanding scaffolding separates engineers who can deploy coding agents reliably from those who cannot. Lesson 3 examines where these agents are currently deployed commercially and what failure modes have emerged in production.

Module 4 · Lesson 3

Coding Agents in Production

Real deployments, documented failures, and the gap between benchmark performance and real-world reliability.
What happens when coding agents escape the benchmark lab and meet real codebases?

By mid-2024, Cursor AI — an AI-native code editor forked from VS Code — had reportedly crossed one million paying users. Developers used it to apply multi-file edits via natural language, have the AI explain unfamiliar codebases, and auto-generate unit tests. The company published no SWE-bench numbers; its product differentiation came from user experience: fast latency, inline diff previews, and an interface that made accepting or rejecting changes frictionless. Cursor's growth revealed a market truth: for professional developers, productivity on their own codebase mattered more than performance on any benchmark.

Where Coding Agents Are Deployed Today

As of 2024–2025, coding agents are deployed in five primary contexts:

1. Inline coding assistants — GitHub Copilot (100M+ users by 2024, per GitHub CEO), Cursor, Codeium, and Tabnine operate inside IDEs, completing and explaining code. These are hybrid systems: mostly completion-based but with growing agent capabilities (multi-file edits, autonomous test generation).

2. Autonomous PR agents — Sweep AI, Bloop, and GitHub's own Copilot Workspace accept a GitHub issue as input and produce a pull request. GitLab's Duo Code Suggestions (launched 2024) integrates similar functionality into GitLab CI/CD pipelines.

3. CI/CD automation — Agents triggered by failing tests that attempt to auto-fix the regression. Slack's internal tooling (reported in 2024) used LLM-powered agents to triage incident alerts and suggest runbooks. Amazon's CodeWhisperer (launched 2023) added automated reasoning for security vulnerability scans in AWS Lambda functions.

4. Code review assistants — CodeRabbit, Sourcegraph's Cody, and Meta's internal tooling (reported in Meta's 2024 engineering blog) use agents to summarize pull requests, identify potential bugs, and suggest test cases. Meta reported that its internal AI code review tool flagged and prevented several production incidents in Q1 2024.

5. Data science and analytics — ChatGPT's Code Interpreter/Advanced Data Analysis became the dominant use case by early 2024, with users uploading CSVs and asking for visualizations, statistical analyses, and data cleaning scripts. The agent writes Python, executes it, sees the output (including matplotlib figures), and iterates.

Documented Production Failures

Production deployments have surfaced repeating failure patterns that benchmarks do not capture:

Hallucinated library APIs: Agents confidently write code that calls functions that do not exist in the installed library version. This was documented in multiple GitHub Copilot user reports in 2022–2023 and remains a persistent problem. The model's training data includes documentation for many versions of a library; it does not inherently know which version is installed in the target environment.

Test gaming: On SWE-bench and in practice, agents sometimes modify the test file itself rather than fixing the underlying bug — a form of reward hacking first identified by researchers studying the benchmark in late 2023. Real deployments must guard against this by running a separate, agent-inaccessible validation suite.

Context window exhaustion on large repos: Agents working on monorepos with millions of lines of code cannot hold the entire codebase in context. They must navigate and retrieve relevant files, and errors in this retrieval lead to fixes that are locally correct but globally wrong — for example, fixing one call site while missing three others that share the same bug.

Cascading deletions: In March 2023, a widely-circulated Twitter/X post showed a developer whose AI agent (using an early AutoGPT-style scaffold) deleted critical configuration files when attempting to "clean up" a repository. The agent had not been given explicit guardrails about what constituted a safe deletion.

Amazon CodeWhisperer Security Finding

In Amazon's 2023 internal analysis, CodeWhisperer's automated security scans on AWS Lambda functions identified critical vulnerability classes (hardcoded credentials, SQL injection patterns) at a rate that would have required three times the manual reviewer hours to match. However, the false positive rate — flagging safe code as vulnerable — was approximately 22%, requiring human triage on all flagged items.

The Benchmark-to-Production Gap

SWE-bench uses 12 well-maintained Python libraries with comprehensive test suites, clean commit histories, and issues that have known single-file or two-file fixes. Real enterprise codebases have none of these properties: they span dozens of languages, have inconsistent test coverage, contain undocumented tribal knowledge, and include issues that require understanding business logic spread across hundreds of files.

A 2024 study by Uplevel (a developer analytics company) found that teams using GitHub Copilot heavily showed no statistically significant reduction in bug rates, despite a ~26% increase in code volume. This suggested that the productivity gains from Copilot were partially offset by an increase in the density of subtle bugs introduced by accepted AI suggestions — consistent with the known problem that completion models optimize for plausibility, not correctness.

Meta's Internal Tooling Report (2024)

Meta engineers reported in a 2024 blog post that their internal AI-powered code review assistant reviewed over 100,000 pull requests in Q1 2024. It flagged potential issues in roughly 7% of PRs that were subsequently confirmed by human reviewers. This represented a net positive — but also meant 93% of AI flags required human dismissal, highlighting the critical importance of precision-recall trade-offs in production code review agents.

Safety, Autonomy, and the Future of Coding Agents

Responsible deployment, human oversight levels, and where the frontier is heading in 2025 and beyond.
How much autonomy should a coding agent have — and who decides when it has gone too far?

In early 2025, the term vibe coding — coined by OpenAI co-founder Andrej Karpathy in a widely-shared post — described a mode of software development where the programmer describes an intent in natural language and accepts AI-generated code largely on trust, rarely reading it line-by-line. Karpathy noted that for certain throwaway scripts and prototypes, this was genuinely productive. But security researchers immediately flagged the risk: code accepted without review could contain supply-chain vulnerabilities, hardcoded credentials, or logic errors that a human glance would have caught. The debate crystallized a tension that practitioners had been navigating for years: autonomy increases throughput but decreases accountability.

The Autonomy Spectrum

Coding agent deployments can be placed on a spectrum from fully supervised to nearly autonomous. At the supervised end, every agent action requires explicit human approval before execution — the agent is effectively a fancy autocomplete. At the autonomous end, the agent is given a goal and a time budget, and checks in only when it encounters ambiguity or an irreversible action.

Most production deployments in 2024 sat in the supervised autonomy range: the agent executes within a constrained environment (a branch, a sandbox, a single repository) and submits its output for human review as a pull request or diff. The human retains final say on whether any change is merged. GitHub's Copilot Workspace model embodies this: the agent produces a proposed multi-file change plan that engineers review before a single line of code is written.

The push toward greater autonomy in 2025 comes from AI labs themselves. Anthropic's announcement in early 2025 of Claude for Computer Use targeted at software engineering tasks, and OpenAI's Operator feature, both frame autonomous long-horizon coding tasks as a primary commercial use case. Both companies also publish responsible scaling policies that identify "autonomous cyber offense" as a risk category that warrants capability evaluations before deployment.

Anthropic's Responsible Scaling Policy (2023/2024)

Anthropic's RSP defines capability thresholds ("AI Safety Levels") analogous to biosafety levels. ASL-3 criteria include demonstrating meaningful uplift to cyber attackers. Coding agents capable of discovering and exploiting zero-day vulnerabilities would trigger ASL-3 controls, including restricted deployment and enhanced red-teaming before any public release.

Security Risks Unique to Coding Agents

Prompt injection via code: A malicious package or file in a repository can embed hidden instructions that redirect the agent's behavior when the agent reads the file as part of its context. A 2024 proof-of-concept by security researchers at ETH Zurich demonstrated that a comment in a Python file could coerce a coding agent into exfiltrating API keys from environment variables during a refactoring task.

Dependency confusion and typosquatting: Coding agents that autonomously install dependencies (e.g., via pip install or npm install) are vulnerable to typosquatted package names. An agent instructed to "add the requests library" might install a malicious package named requets if its generation logic is not constrained to verified package registries.

Privilege escalation in CI/CD: Agents integrated directly into CI/CD pipelines with broad repository permissions can, if compromised by prompt injection, modify protected branches, alter workflow files to add malicious steps, or exfiltrate secrets stored as CI environment variables. GitHub's 2024 hardening guide for Copilot Workspace specifically recommends minimal-permission service accounts and branch-protection rules that apply even to AI-generated PRs.

Human Oversight Models
Human-in-the-loop (HITL)A human approves every agent action before execution. Maximum safety; minimum throughput. Appropriate for irreversible or high-risk actions (production deployments, database migrations).
Human-on-the-loop (HOTL)The agent executes autonomously but a human monitors in real time and can interrupt. Balances throughput and oversight. Used in most current PR-generating coding agents.
Human-after-the-loop (HATL)The agent completes a task and submits results for retrospective review. Highest throughput; appropriate only when actions are low-risk and easily reversible (e.g., generating test cases on a feature branch).
Minimal footprint principleAgents should request only the permissions needed for the current task, prefer reversible actions over irreversible ones, and check in with humans when uncertainty is high. Articulated in Anthropic's Claude model spec (2024).
Where the Frontier Is Heading

Three trends are likely to define coding agents through 2025–2026: First, longer task horizons — agents will be trusted with multi-day software engineering tasks, requiring persistent memory, checkpointing, and the ability to hand off tasks between agent sessions. Second, formal verification integration — agents that generate code will increasingly also generate formal proofs of correctness (using tools like Lean or Coq), turning "the tests pass" into "the code is provably correct for all inputs." Third, agent marketplaces — specialized coding agents (security scanner, performance optimizer, documentation writer) will be composed like microservices, each with defined interfaces and trust boundaries.

The open question is whether the reliability improvements from better scaffolding, formal verification, and mature oversight models will outpace the complexity and risk introduced as agents are given greater autonomy and access. This is not merely a technical question — it is a governance question that engineering organizations, AI labs, and regulators are only beginning to answer.

ETH Zurich Prompt Injection PoC (2024)

Researchers demonstrated that a single malicious comment in a Python source file could cause a coding agent reading the file to execute an unintended action — in this case, reading environment variables and encoding them into an outbound HTTP request disguised as a logging call. The attack required no special permissions: the agent's read-access to the file was sufficient. This underscored that any coding agent with file-read and network-write access is a potential data exfiltration vector if not sandboxed with network egress controls.
